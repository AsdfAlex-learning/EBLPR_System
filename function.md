# 函数总数与分类统计

这份代码总共包含 21 个函数，可分为四大类：主函数、预处理 / 特征检测函数、场景识别算法函数、通用识别工具函数。下面按类别列出各函数及其功能说明。

---

## 一、主函数（1 个）

| 函数名 | 功能 |
|---|---|
| `process_image` | 车牌识别主入口：接收图像路径，完成图像预处理、车牌定位裁剪、图像类型检测，根据类型调用对应识别算法，最终输出裁剪后的车牌路径和识别出的车牌号码。 |

---

## 二、预处理 / 特征检测函数（7 个）

| 函数名 | 功能 |
|---|---|
| `preprocess_image` | 图像基础预处理：对灰度图做直方图均衡化增强对比度、中值滤波去噪、自适应二值化、形态学开闭运算去除小噪点。 |
| `detect_image_type` | 图像类型检测：结合倾斜角度、车牌数量、干扰程度，将图像分为 normal（正常）、tilted（倾斜）、interfered（干扰）、multiple（多车牌）四类。 |
| `detect_tilt_angle` | 倾斜角度检测：通过 Canny 边缘检测 + Hough 变换，计算图像中主要直线的倾斜角度。 |
| `detect_plate_count` | 车牌数量检测：连通域分析 + 面积过滤，统计图像中符合车牌面积特征的区域数量。 |
| `detect_interference` | 干扰程度检测：通过计算图像梯度的方差，量化图像的干扰程度（值越大干扰越强）。 |
| `remove_interference` | 干扰去除：形态学开运算 + 连通域面积过滤 + 中值滤波，去除图像中的小干扰文字 / 图案。 |
| `enhance_after_cleaning` | 去干扰后增强：对比度调整 + 形态学闭运算，修复去干扰后字符的断裂问题。 |

---

## 三、场景识别算法函数（4 个）

| 函数名 | 功能 |
|---|---|
| `recognize_normal_plate` | 正常车牌识别：对清晰无干扰的车牌，完成增强、定位、字符分割、字符识别。 |
| `recognize_tilted_plate` | 倾斜车牌识别：先检测并矫正倾斜角度（≤15°），再调用正常车牌识别流程。 |
| `recognize_interfered_plate` | 干扰车牌识别：先去除干扰、增强图像，再调用正常车牌识别流程。 |
| `recognize_multiple_plates` | 多车牌识别：检测所有车牌区域，选择“面积 × 清晰度”得分最高的目标车牌，再调用正常车牌识别流程。 |

---

## 四、通用识别工具函数（9 个）

| 函数名 | 功能 |
|---|---|
| `enhance_normal_plate` | 正常车牌增强：对比度调整 + 轻微锐化，提升清晰车牌的字符辨识度。 |
| `detect_all_plates` | 多车牌区域检测：连通域分析 + 面积 / 宽高比 / 紧凑度过滤，提取所有疑似车牌区域。 |
| `select_target_plate` | 目标车牌选择：计算每个车牌区域的“清晰度 × 面积”得分，选择最高分的区域作为目标。 |
| `extract_plate_region` | 车牌区域提取：根据 BoundingBox 裁剪出车牌区域（做边界保护，避免越界）。 |
| `locate_plate_region` | 单车牌定位：连通域分析 + 评分（面积 + 宽高比 + 紧凑度），找到最符合车牌特征的区域。 |
| `segment_characters` | 字符分割：垂直投影法找到字符列边界，分割出单个字符（过滤过小区域，去除字符上下空白）。 |
| `recognize_characters` | 字符批量识别：遍历分割后的字符，调用模板匹配函数，拼接识别结果（过滤全 “?” 的无效结果）。 |
| `load_char_templates` | 字符模板加载：从指定文件夹加载 0-9、A-Z、“粤” 的二值化模板，统一尺寸为 40×20 并存储到全局变量。 |
| `match_character` | 单字符匹配：将待识别字符归一化尺寸后，与模板做归一化互相关计算，返回相似度最高的字符（低于 0.6 阈值返回 “?”）。 |

---

## 总结

- **核心逻辑**：`process_image` 作为入口，先预处理图像，再检测图像类型，最后调用对应场景的识别算法。所有场景最终都依赖“定位 → 分割 → 模板匹配”的通用流程完成识别。
- **函数分工**：预处理 / 特征检测函数负责图像前期分析，场景算法函数适配不同车牌状态，通用工具函数实现识别的核心操作（定位、分割、模板匹配）。
- **关键依赖**：字符识别基于模板匹配实现，需提前准备字符模板文件，否则会触发错误提示。
